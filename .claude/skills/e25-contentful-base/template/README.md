# Contentful Next.js Starter

Next.js + React + Tailwind + Contentful scaffolding, generated by the
`e25-contentful-base` skill. The base scaffold comes from
`create-next-app@latest`; the Contentful integration is overlaid on top.

What's installed:

- **Catch-all route** that fetches a `FlexiblePage` from Contentful and renders
  its sections (no sections shipped — add them per project).
- **SEO metadata** wired into Next.js `generateMetadata`.
- **Locale-aware routing** (English + Spanish by default).
- **Section registry** scaffolding — you add concrete sections.
- **Brand tokens** as Tailwind 4 CSS variables in `app/globals.css`.

No Header, Footer, Breadcrumbs, or example sections ship — this is pure
scaffolding.

## Architecture

Read [`components/ARCHITECTURE.md`](./components/ARCHITECTURE.md) for the full
explanation of how Contentful entries become rendered components, plus the
six-step recipe for adding a new section.

## Getting started

```bash
cp .env.example .env.local         # then fill in your Contentful credentials
npm run dev
```

Open <http://localhost:3000>.

## Environment variables

| Variable                          | Required | Default                    |
| --------------------------------- | -------- | -------------------------- |
| `CONTENTFUL_SPACE_ID`             | yes      | —                          |
| `CONTENTFUL_DELIVERY_TOKEN`       | yes      | —                          |
| `CONTENTFUL_ENVIRONMENT`          | no       | `master`                   |
| `CONTENTFUL_PREVIEW_TOKEN`        | no       | —                          |
| `CONTENTFUL_GRAPHQL_HOST`         | no       | `graphql.contentful.com`   |
| `CONTENTFUL_PREVIEW_GRAPHQL_HOST` | no       | `preview.contentful.com`   |
| `CONTENTFUL_MANAGEMENT_TOKEN`     | no       | — (only used by `scripts/seed-contentful.mjs`; **write access**) |

⚠️ The `CONTENTFUL_MANAGEMENT_TOKEN` has full write access to your space. The
Next.js runtime never reads it, but it lives in `.env.local` for convenience
when re-running the seed script. Do not commit `.env.local`, do not include it
in deploy bundles, and revoke the token if it ever leaks.

## Contentful content models

Minimum required to make the catch-all route work:

- **FlexiblePage** — `slug`, `pageTitle`, `seo` (ref to Seo), `sections`
  (multi-ref to your section entries)
- **Seo** — `seoTitle`, `seoDescription`, `seoOgImage` (Asset), `seoNoIndex`,
  `seoNoFollow`, `seoCanonicalUrl`, `seoSchemaMarkup`

The scaffold ships `scripts/seed-contentful.mjs` which creates both content
types and a sample `/` entry. To re-run it later (after a space reset, or to
sync the schema):

```bash
node scripts/seed-contentful.mjs
```

The script reads `CONTENTFUL_SPACE_ID`, `CONTENTFUL_ENVIRONMENT`, and
`CONTENTFUL_MANAGEMENT_TOKEN` from `.env.local` (resolved relative to the
script's own location, so any CWD works). Generate a CMA token at
Contentful → Settings → API keys → Content management tokens.

Section content models are added per project; see the architecture doc.
