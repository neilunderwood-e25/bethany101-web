# Contentstack Next.js Starter

Next.js + React + Tailwind + Contentstack scaffolding, generated by the
`e25-contentstack-base` skill. The base scaffold comes from
`create-next-app@latest`; the Contentstack integration is overlaid on top.

What's installed:

- **Catch-all route** that fetches a `flexible_page` from Contentstack and
  renders its sections (no sections shipped — add them per project).
- **SEO metadata** wired into Next.js `generateMetadata`.
- **Locale-aware routing** (English + Spanish by default).
- **Section registry** scaffolding — you add concrete sections, each of which
  contributes a GraphQL fragment that is composed into the page query.
- **Brand tokens** as Tailwind 4 CSS variables in `app/globals.css`.

No Header, Footer, Breadcrumbs, or example sections ship — this is pure
scaffolding.

## Architecture

Read [`components/ARCHITECTURE.md`](./components/ARCHITECTURE.md) for the full

## Getting started

```bash
cp .env.example .env.local         # then fill in your Contentstack credentials
npm run dev
```

Open <http://localhost:3000>.

## Environment variables

| Variable                            | Required | Default                            |
| ----------------------------------- | -------- | ---------------------------------- |
| `CONTENTSTACK_API_KEY`              | yes      | —                                  |
| `CONTENTSTACK_DELIVERY_TOKEN`       | yes      | —                                  |
| `CONTENTSTACK_ENVIRONMENT`          | yes      | —                                  |
| `CONTENTSTACK_BRANCH`               | no       | — (stack default branch)           |
| `CONTENTSTACK_PREVIEW_TOKEN`        | no       | —                                  |
| `CONTENTSTACK_GRAPHQL_HOST`         | no       | `graphql.contentstack.com`         |
| `CONTENTSTACK_PREVIEW_GRAPHQL_HOST` | no       | `graphql-preview.contentstack.com` |
| `CONTENTSTACK_API_HOST`             | no       | `api.contentstack.io` (seed only)  |
| `CONTENTSTACK_MANAGEMENT_TOKEN`     | no       | — (only used by the seed; **write access**) |

The delivery token is environment-specific and rides in the `access_token`
header of the GraphQL Content Delivery API. For non-US regions, change the
`*_HOST` values (see `.env.example` for the EU / Azure / GCP prefixes).

⚠️ The `CONTENTSTACK_MANAGEMENT_TOKEN` has write access to your stack. The
Next.js runtime never reads it, but it lives in `.env.local` for convenience
when re-running the seed script. Do not commit `.env.local`, do not include it
in deploy bundles, and revoke the token if it ever leaks.

## Contentstack content models

Minimum required to make the catch-all route work:

- **flexible_page** — `title`, `url` (unique), `seo` (reference to `seo`).
  Add a `sections` multiple-reference field once you build your first section.
- **seo** — `title`, `description`, `og_image` (file), `no_index`,
  `no_follow`, `canonical_url`, `schema_markup`

The scaffold ships `scripts/seed-contentstack.mjs`, which creates both content
types and a sample `/` entry. To re-run it later (after a stack reset, or to
sync the schema):

```bash
node scripts/seed-contentstack.mjs
```

The script reads `CONTENTSTACK_API_KEY`, `CONTENTSTACK_ENVIRONMENT`,
`CONTENTSTACK_API_HOST`, and `CONTENTSTACK_MANAGEMENT_TOKEN` from `.env.local`
(resolved relative to the script's own location, so any CWD works). Generate a
management token at Contentstack → Settings → Tokens → Management Tokens.

Section content models are added per project; see the architecture doc.
